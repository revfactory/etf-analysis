# ETF Data Research

코스닥 ETF 시장과 펀드별 포트폴리오를 체계적으로 수집하는 방법론.

## 수집 원칙

공식 소스 우선순위를 엄격히 따른다. 추정은 금지.

1. **운용사 공시** — 삼성자산운용, 미래에셋, KB, 한화, NH-Amundi 등의 ETF 월간 운용보고서
2. **KRX 정보데이터시스템** (data.krx.co.kr) — ETF 공시, 일별 구성 종목(PDF)
3. **ETF Check / 에프앤가이드** — 시계열 비중 변화 (무료 API 일부)
4. **네이버 금융 / 인베스팅닷컴** — 보조 교차 확인용
5. **언론 보도** — 운용보고서 요약/해설 (최후 수단)

## ETF 상위 3개 선정 기준

**기본 기준: 순자산(AUM)** — 사용자가 별도 지정하지 않으면 AUM 기준 Top 3.

사용자가 요청하는 "상위"의 모호함 해소:
- "상위" = 순자산 기준 (default)
- 거래량 기준 요청 시 → 일평균 거래대금
- 수익률 기준 요청 시 → 1년/3년 수익률

## 시점 선정

기본: 최신 공시일, 3개월 전, 6개월 전 — 3개 시점으로 변화 추적.
추가 확장 가능: 12개월 전 시점 포함 (데이터 입수 가능 시).

## 데이터 스키마

### ETF 메타데이터 (`01_etf_top3.json`)
```json
{
  "collected_at": "2026-04-14",
  "etfs": [
    {
      "code": "091160",
      "name": "KODEX 코스닥150",
      "asset_manager": "삼성자산운용",
      "underlying_index": "코스닥150",
      "aum_krw_bn": 1234.5,
      "listed_at": "2015-10-01",
      "expense_ratio": 0.25,
      "aum_rank": 1
    }
  ]
}
```

### Holdings (`01_holdings_{code}.json`)
위 agent 정의 참조. 각 종목은 `ticker`, `name`, `sector`(선택), `weights`(시점별), `weight_change_pct`, `rank_latest`, `is_new`(신규 편입), `is_removed`(제외) 필드 포함.

## 수집 절차

1. **AUM 순위 확인** — KRX ETF 공시 또는 언론의 "ETF AUM 순위" 기사 교차 확인
2. **상위 3개 선정 및 메타데이터 수집** (`01_etf_top3.json`)
3. **각 ETF별로 3개 시점 PDF 공시 다운로드 및 파싱**
4. **종목 데이터 정규화** (티커 일치 확인, 리네이밍 이력 매핑)
5. **비중 변화 계산** → holdings JSON에 포함
6. **출처 기록** (`01_sources.md`): 각 데이터 항목의 URL + 접근 일자

## 주의사항

- **공시 기준일**은 종종 발표일과 다름 (발표 T+N). JSON의 `as_of_dates`는 실제 기준일을 사용
- **레버리지/인버스 ETF 제외** — 피투자사 개념이 없거나 파생 포지션
- **합병/분할**로 티커가 바뀐 종목은 `ticker_history` 필드로 이력 보존
- **데이터 접근 실패 시 N/A** — 추정치 절대 금지

## 실제 수집 방법 (WebFetch/WebSearch 기반)

공식 데이터 접근이 제한적일 때:
1. WebSearch로 "KODEX 코스닥150 구성 종목 2026" 등 검색
2. 발견된 공식 운용사 페이지 → WebFetch
3. 언론 보도 교차 확인 (조선비즈, 매일경제, 아시아경제 ETF 코너)
4. 데이터 표가 HTML에 있으면 직접 파싱, PDF면 다운로드 후 텍스트 추출

## 한계 명시

- 일부 운용사는 전체 종목 리스트를 월 1회만 공시 → 일간 변화는 추적 불가
- 비공시 소수 종목은 "기타" 범주로 합산
- 실시간 변동 (당일 리밸런싱)은 반영 불가

이러한 한계는 반드시 `01_sources.md` 상단에 명시한다.
